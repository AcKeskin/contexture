# Register module — technical / developer docs

README, API docs, guides, changelogs, design docs. Imports the catalogue (`ai-vocabulary.v1.md`) + carve-out (`false-positives.md`); this module adds register specifics.

## Register-specific tells (flag by density)

- **Significance puffery on a tool** — "a powerful, flexible, and intuitive framework", "revolutionizing how developers…". Dev docs describe mechanics, not destiny.
- **Hollow feature triplets** — stacked rule-of-three adjectives in feature lists ("fast, reliable, and scalable") with no numbers behind them.
- **"In today's fast-paced development landscape…"** openers and "Despite these challenges…" closers.
- **Uniform section rhythm** — every section the same length, every bullet the same shape, no asymmetry where the hard part deserves more.
- **Sycophantic onboarding** — "Great! Now that you've installed it…".

## Conventions to PRESERVE (never flag — native here)

- Headers, sub-headers, **boldface**, bulleted/numbered lists, tables, code fences. Structural; not anomalies.
- Em-dashes, parentheticals, colons introducing lists.
- Imperative mood ("Run", "Clone", "Add the import line").
- A single rule-of-three when it genuinely enumerates three things.
- Consistent terminology (dev docs *want* the same term for the same concept — that is not "synonym-avoidance" or AI uniformity).
